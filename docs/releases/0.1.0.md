# Solodeveling 0.1.0

Solodeveling 0.1.0 is the first alpha candidate of the single-agent-first software
delivery protocol and portable skill suite. One public package and command name,
`solodeveling`, covers Codex, Claude Code, Cursor, and generic Agent Skills
clients without requiring subagents.

## Install and operate

The candidate is not on npm or PyPI yet. After the reviewed release is authorized and
published, Node.js users can start without Python:

~~~console
npx solodeveling install
~~~

Persistent Node and isolated Python options use the same command:

~~~console
npm install -g solodeveling
uv tool install solodeveling
pipx install solodeveling
~~~

Use `solodeveling check` after installation and upgrades. Remove unchanged managed
files with `solodeveling uninstall`; modified files are preserved.

## Included capabilities

- Ten canonical skills for routing, onboarding, shaping, planning, execution,
  debugging, verification, security, release, and maintenance.
- Persistent, schema-validated project memory and evidence records.
- Collision-, containment-, drift-, symlink-, and uninstall-safe runtime installation.
- Unified install, check, uninstall, init, validate, eval, and version subcommands.
- Python wheel/source distributions and dependency-free npm launcher preparation.
- Native build targets for Windows, macOS, and Linux on x64 and arm64.
- Checksums, installed smoke, CycloneDX SBOM, and provenance gates.
- Deterministic complete cross-ecosystem release set assembly that binds Python, npm,
  and six native targets to one exact source revision without publishing.

## Security and compatibility

The Python distribution requires Python 3.10 or newer. The npm launcher requires
Node.js 20 or newer and has no dependency or install lifecycle script. It downloads
only on explicit invocation, binds an exact release asset, verifies byte size and
SHA-256, rejects symlinked or tampered cache entries, and executes without a shell.

Native support is claimed for a version only after all six platform builds and smoke
tests pass. Checksums alone do not prove publisher identity; verify release provenance
and review `SECURITY.md` before reporting a vulnerability.

## Known limitations

Tier 1 remains unverified. One representative live scenario passed on Codex and
Claude Code, while Cursor live behavior and the complete behavioral matrix remain
unverified. This candidate has not been tagged, attested, released, uploaded to npm
or PyPI, or observed in downstream projects. Registry availability, dependency
vulnerabilities, native CI evidence, artifact identity, and support claims must be
rechecked immediately before publication.
