# Solodeveling 0.1.0

Solodeveling 0.1.0 is the first alpha release candidate of the single-agent-first
software-delivery protocol and portable skill suite. It supports package-driven
adapter installation for Codex, Claude Code, Cursor, and generic Agent Skills
clients without requiring subagents.

## Install and operate

The candidate is not on PyPI yet. Verify the published SHA-256 checksums before
installing the wheel from the authorized GitHub Release. Initialize project memory
with `solodeveling-init`, then install a runtime adapter with
`solodeveling-adapt install --runtime <runtime> --project <path>`. Use
`solodeveling-adapt check` after installation and upgrades. Preview removal with
`solodeveling-adapt uninstall --dry-run`, then run `solodeveling-adapt uninstall`
only after reviewing the affected paths.

## Included capabilities

- Ten canonical skills covering routing, onboarding, shaping, planning, execution,
  debugging, verification, security, release, and maintenance.
- Persistent, schema-validated project memory and evidence records.
- Safe multi-runtime adapter installation with collision, containment, drift,
  symlink, and uninstall controls.
- A bounded cross-agent evaluation harness and non-publishing release builder.
- Wheel and source distribution inventory checks, checksums, installed smoke tests,
  CycloneDX SBOM preparation, and provenance gates.

## Security and compatibility

The package requires Python 3.10 or newer and is tested on Python 3.10 and 3.14
across Windows, Ubuntu, and macOS. Release artifacts include SHA-256 integrity
records; checksums alone do not prove publisher identity. Review `SECURITY.md`
before reporting a vulnerability and do not place secrets in project memory.

## Known limitations

Tier 1 remains unverified. One representative live scenario passed on Codex and
Claude Code, while Cursor live behavior and the complete behavioral matrix remain
unverified. The release candidate has not been tagged, attested, signed, uploaded to
PyPI, or observed in downstream user projects. Recheck package-name availability,
dependency vulnerabilities, artifact identity, and support claims immediately before
publication.
