# Solodeveling 0.1.2

Solodeveling 0.1.2 is a compatible alpha patch release that makes small and ordinary
work faster while retaining the full audited boundary for security, production,
migration, and release work.

## Install and operate

Node.js users can install into the current project without installing Python:

~~~console
npx solodeveling install
~~~

Persistent Node and isolated Python options expose the same command:

~~~console
npm install -g solodeveling
uv tool install solodeveling
pipx install solodeveling
~~~

Run `solodeveling check` after installation or upgrade. Use
`solodeveling uninstall` to remove only unchanged managed files.

## Changes since 0.1.1

- Defined an explicit zero-artifact Ephemeral Quick path: inspect, edit, focused
  verification, and report without WORK, EVIDENCE, state, or roadmap writes.
- Bounded ordinary Standard work to one WORK item and one cumulative EVIDENCE file.
- Reused the existing work/evidence pair for same-boundary verification and small
  corrective follow-ups.
- Limited roadmap updates to actual priority, milestone, ordering, or deferred-work
  decisions and assigned each durable fact one memory owner.
- Added `solodeveling work evidence`, `transition`, and `archive` commands to
  apply validated lifecycle changes with fewer manual edits.
- Added atomic rollback, safe identifier validation, synchronized next actions,
  current-state reconciliation, and stale archive cleanup.
- Prevented a latest failed or unverified evidence summary from transitioning work to
  done, while retaining prior observations in the cumulative evidence body.

## Security and compatibility

The public command, project-memory schema, installation behavior, and supported
Python and Node versions remain compatible with 0.1.1. Critical, security, release,
migration, production, destructive, and sensitive work still uses the full audited
lifecycle with security, recovery, provenance, and authorization boundaries.

The complete cross-ecosystem release set remains bound to one source revision: two
Python distributions, six native executables, one npm tarball, one CycloneDX SBOM,
release notes, SHA-256 checksums, a release-set manifest, and GitHub attestations.

## Known limitations

Tier 1 remains unverified. Codex and Claude Code have each passed one bounded live
scenario; Cursor has structural adapter evidence only. Native executables are not
platform code-signed. Pilot-4 remains blocked by the missing external Codex Windows
sandbox helper, so Solodeveling still makes no controlled cross-framework speed or
quality superiority claim.
