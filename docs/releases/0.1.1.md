# Solodeveling 0.1.1

Solodeveling 0.1.1 is a compatible alpha patch release focused on a clearer first
experience. It corrects the public npm and PyPI landing content, preserves one-command
installation, and keeps small work lightweight while retaining explicit controls for
critical delivery and publication.

## Install and operate

Node.js users can install into the current project without installing Python:

~~~console
npx solodeveling install
~~~

Persistent Node and isolated Python options expose the same command:

~~~console
npm install -g solodeveling
uv tool install solodeveling
pipx install solodeveling
~~~

Run solodeveling check after installation and upgrades. Use solodeveling uninstall
to remove only unchanged managed files.

## Changes since 0.1.0

- Simplified project memory so safe Quick work can remain ephemeral and Standard
  work keeps compact resumable state without unnecessary ceremony.
- Added focused memory-only CI and a bounded docs-only path while retaining the full
  matrix for product, skill, root README, and ambiguous changes.
- Clarified the public README with neutral comparisons, intended users, evidence-backed
  workflow-overhead observations, and honest benchmark limits.
- Replaced stale pre-publication npm and PyPI descriptions with current installation,
  product, integrity, and support guidance.
- Removed a hard-coded package version from the npm CI smoke path and added regressions
  for registry-facing content and cross-ecosystem version agreement.
- Preserved the complete cross-ecosystem release set boundary across Python, npm, six
  native targets, checksums, SBOM, source identity, and attestations.

## Security and compatibility

The public command, project-memory schema, and installation behavior remain compatible
with 0.1.0. Python 3.10 or newer and Node.js 20 or newer are required. The dependency-free
npm launcher downloads only on explicit invocation, selects the version-matched GitHub
Release asset, verifies size and SHA-256, rejects unsafe cache entries, and executes
without a shell.

## Known limitations

Tier 1 remains unverified. Codex and Claude Code have each passed one bounded live
scenario; Cursor has structural adapter evidence only. Native executables are not
platform code-signed. The comparison describes documented defaults and measured local
workflow overhead, not a controlled cross-framework speed or quality benchmark.
