# Solodeveling 0.2.0

Solodeveling 0.2.0 is an alpha feature release that makes intent routing explicit,
keeps iterative work lightweight between checkpoints, and classifies backend risk by
the boundary changed rather than by technology presence.

## Install and operate

Node.js users can install into the current project without installing Python:

~~~console
npx solodeveling install
~~~

Persistent Node and isolated Python options expose the same command:

~~~console
npm install -g solodeveling
uv tool install solodeveling
pipx install solodeveling
~~~

Run `solodeveling check` after installation or upgrade. Use
`solodeveling uninstall` to remove only unchanged managed files.

## Changes since 0.1.2

- Added Direct Read-Only routing before project-memory reads and persistence tiers.
  Ordinary Q&A, explanation, review, status, diagnosis, and inline advice inspect
  only what their claims require and create no lifecycle artifacts by default.
- Added `solodeveling-brainstorming` for open-ended project exploration before stack,
  specification, or project-memory commitment.
- Made Ephemeral Quick take precedence over reopening or reusing prior tracked work
  for bounded same-session follow-ups.
- Added same-surface frontend iteration batches with focused checks per slice and
  broad affected gates once at commit, handoff, changed boundary, or closure.
- Classified server-side and backend work by changed boundary and produced effect.
  Authenticated routes are not Critical merely because authentication is present;
  credential, session, role, permission, ownership, payment-fulfillment, sensitive,
  and trust-boundary changes retain the Critical path.
- Added checkpoint-valid verification, one Current acceptance matrix, supersession
  markers, concise observation logs, and code-level versus release-level provider
  evidence.
- Added non-interactive command-profile guidance and behavioral scenarios for
  read-only, brainstorming, frontend batching, backend routing, and security matrices.
- Added complete English and Thai README documentation with language navigation, and
  included both documents in the Python source distribution.

## Security and compatibility

The public CLI, project-memory schema version 1, installation behavior, and supported
Python and Node versions remain compatible with 0.1.2. Routing behavior intentionally
changes, so agents should reload the installed 0.2.0 skills before comparing results.

Critical work still requires scoped security and recovery evidence. The release does
not lower credential, session, authorization-decision, payment-fulfillment,
sensitive-data, destructive migration, production, secret, cryptographic, or
safety-critical boundaries.

The complete cross-ecosystem release set remains bound to one exact source revision:
two Python distributions, six native executables, one npm tarball, one CycloneDX
SBOM, release notes, SHA-256 checksums, a release-set manifest, and GitHub
attestations.

## Known limitations

Tier 1 remains unverified. The new routing has scenario and blind forward-test
coverage but limited longitudinal field evidence. Native executables are not platform
code-signed. Solodeveling still makes no controlled cross-framework speed or quality
superiority claim.
