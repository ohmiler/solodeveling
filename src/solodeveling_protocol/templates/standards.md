# Standards

Record the project Definition of Done and coding, testing, security, UX, and
operational conventions. Link existing authoritative instructions instead of
copying them.

Record canonical non-interactive verification commands and their scope. Distinguish
watch-mode developer commands from run-once commands suitable for agents and CI.
Include focused, affected-regression, lint, build, and broad-gate commands only when
the repository confirms them.
